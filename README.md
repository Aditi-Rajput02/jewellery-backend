# BG Jewels Search API

A FastAPI backend for **BG Jewels** — an AI-powered jewellery search and catalog management system. It combines **OpenAI GPT-4o Vision**, **CLIP embeddings**, **FAISS vector search**, **ChromaDB RAG**, and **MongoDB Atlas** to deliver intelligent image-based and text-based jewellery search.


## Features

- 🔍 **AI Image Search** — Upload a jewellery photo; GPT-4o Vision describes it, then CLIP + FAISS finds the closest matches in the catalog.
- 💬 **Natural Language Catalog Filtering** — Parse free-text queries (e.g. *"gold necklace under ₹50000"*) into structured filters using GPT-4o-mini with a regex fallback.
- 🧠 **RAG Knowledge Base** — Store jewellery descriptions as CLIP-embedded transcripts in ChromaDB and retrieve the most relevant ones by semantic similarity.
- 🖼️ **Image Proxy** — Bypass hotlink protection on external image CDNs by proxying requests server-side.
- 📦 **Catalog Management** — Filter, paginate, and update product descriptions across MongoDB collections.
- 🗂️ **Search History** — Every search event is persisted to MongoDB, and matched SKU descriptions are auto-updated.

## Installation

1. **Clone the repository**

   ```bash
   git clone <repo-url>
   cd backend
   ```

2. **Create and activate a virtual environment**

   ```bash
   python -m venv venv
   # Windows
   venv\Scripts\activate
   # macOS / Linux
   source venv/bin/activate
   ```

3. **Install dependencies**

   ```bash
   pip install -r requirements.txt
   ```

4. **Install OpenAI CLIP** (not on PyPI — install directly from GitHub)

   ```bash
   pip install git+https://github.com/openai/CLIP.git
   ```

5. **Build the FAISS index** (required for `/search` endpoint)

   ```bash
   python build_index.py
   ```

   This generates `index.faiss` and `metadata.json` from your MongoDB catalog.

## API Endpoints

### Search

#### `POST /search`
Upload a jewellery image to find similar products.

**Pipeline:** Image → GPT-4o Vision description → CLIP text encoder → FAISS search

| Parameter | Type | Description |
|---|---|---|
| `image` | file | Jewellery image (JPEG, PNG, etc.) |
| `top_k` | int | Number of results to return (1–20, default: 8) |

## How It Works

### Image Search Pipeline

```
1. User uploads a jewellery image
        │
        ▼
2. GPT-4o Vision analyses the image
   → generates a natural-language description
   e.g. "A yellow gold necklace with polki diamonds and emerald drops"
        │
        ▼
3. CLIP text encoder converts the description → 512-dim embedding vector
        │
        ▼
4. FAISS searches the pre-built index for the nearest vectors
   (catalog was indexed using CLIP text embeddings of product descriptions)
        │
        ▼
5. Top-K matching products returned with confidence scores
        │
        ▼
6. Search event + matched SKUs saved to MongoDB search_history
```

### RAG Pipeline

```
SAVE:
  description text → CLIP text encoder → 512-dim vector
  → ChromaDB upsert (id = sku_name or UUID)

SEARCH:
  query text → CLIP text encoder → 512-dim vector
  → ChromaDB cosine similarity search
  → top-K transcripts ranked by similarity
```

The RAG pipeline uses the **same CLIP embedding space** for both saving and searching, ensuring semantic consistency. Confidence scores are scaled to a 40–100% range for display.

## Database

### MongoDB Collections

| Collection | Description |
|---|---|
| `catalog` | Core product catalog (SKU, metal, item type, price, stock, images) |
| `product_details` | Extended product info (stone list, weights, pricing breakdown) |
| `product_list` | Additional product metadata |
| `search_history` | Log of all search events with matched SKUs and descriptions |

### ChromaDB

- **Location:** `./rag_chroma_db/` (auto-created)
- **Collection:** `rag_transcripts`
- **Similarity metric:** Cosine similarity
- **Embedding dimension:** 512 (CLIP ViT-B/32)

### FAISS Index

- **Files:** `index.faiss` + `metadata.json`
- **Built by:** `build_index.py` (run once, or whenever the catalog changes)
- **Embedding:** CLIP text embeddings of product descriptions
